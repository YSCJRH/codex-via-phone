# v0.1.0 - First Public Pre-Release

## Version overview

- Version: `v0.1.0`
- Release date: 2026-04-18
- Status: pre-release

## Highlights

- real-device verified mobile UI across iPhone Safari, iOS PWA, and Android Chrome
- new README homepage visuals, including a mobile hero collage and a sanitized real-device preview gallery
- refined mobile home, chat, search, approval, permission, and empty-state flows for the public mirror

## Install / upgrade notes

- Supported upstream version: `siteboon/claudecodeui v1.25.2`
- Re-apply override layer required: yes, if you are upgrading an existing upstream checkout
- Re-run `npm install` required: only if your local upstream tree or dependencies are missing
- Re-package desktop tool required: no packaged binary is included in this pre-release

## Security notes

- This release does not expand the default boundary. `localhost` remains the default mode.
- `tailnet-private` remains the recommended remote mode, and `public-funnel` remains explicit and dangerous.
- Desktop approval, device-key trust, and redacted support/export defaults remain unchanged.
- README screenshots are sanitized public docs assets, not raw runtime evidence.

## Documentation

- README updated: yes
- Deployment guide updated: no
- Security doc updated: no
- Real-device test checklist added: yes

## Known issues

- public screenshots were sanitized from real-device captures and should still be re-reviewed on the GitHub web page after release
- `scripts/smoke-test-override-flow.ps1` still requires an upstream zip in `vendor/` or an explicit `-UpstreamZip` path

## Upgrade guidance

- If you already run the project locally, pull the new README/docs changes and re-apply the override layer to your upstream tree if needed.
- For a fresh setup, start in `localhost`, validate locally, then opt into `tailnet-private` only if you actually need phone access.
- If you publish a fork, keep the screenshot assets inside the explicit README allowlist and do not add raw screenshots elsewhere in the tree.
