# 私有本地内容排除清单

[中文](PRIVATE_LOCAL_ONLY.zh-CN.md) | [English](PRIVATE_LOCAL_ONLY.md)

这份文档用于列出哪些内容只应该留在本地工作副本里，不能进入公开仓库快照。

## 可以保留在公开仓库里的内容

公开仓库只保留最小必要的可复用材料：

- 覆盖层源码
- 辅助脚本
- 桌面控制工具源码
- 部署、安全、贡献和发版审查所需文档

## 必须排除在公开仓库之外的内容

以下内容不要发布：

- `.runtime/`、`tmp/`、日志、数据库、诊断导出等运行时状态
- `vendor/` 中的完整上游源码快照
- 打包好的二进制、本地压缩包和构建残留
- 证书、私钥和个人 TLS 材料
- session JSONL、审批痕迹、可信设备证据以及从运行态截取的截图
- 真实主机名、tailnet 域名、私有 IP、用户名和机器专属绝对路径
- 只对某一次私人部署有意义、但对公开项目没有复用价值的说明

## 统一改成占位符

如果文档或示例必须提到本地值，请统一改成占位符，例如：

- `https://mobile-codex.example.com`
- `<PRIVATE_HTTPS_ENTRYPOINT>`
- `<TAILNET_IP>`
- `<PATH_TO_MOBILE_CODEX_HELPER>`

## 历史诊断文档

历史分析文档只有在完成公开安全改写后，才适合继续保留：

- 去掉真实域名和 IP
- 去掉和私人设备或审批窗口绑定的具体时间点
- 去掉复制出来的运行时证据和会话细节
- 只保留可复用的方法、故障模式和部署经验

如果某个文件读起来仍然像“私人事故复盘”，那它在完成脱敏前就应该被视为私有本地内容。
